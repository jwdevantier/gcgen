The code emitter¶

All snippets and generators are provided an emitter, which they must use to generate their output.

The emitter provides a convenient if simple abstraction for output generation. The main objective of the API is to control lines and indentation. This permits gcgen to ensure that output generated from a snippet is always properly indented, such that each line of the snippet are indented relative to the indentation of the snippet start line.

Example¶

from gcgen.api import Scope, Emitter, Json, snippet

@snippet("my-snippet")
def include_file_output(e: Emitter, s: Scope, v: Json):
    if isinstance(v, str):
        filename = v
    else:
        raise RuntimeError("expected filename passed as string argument")
    e.emitln(f"""with open({filename!r}) as fh:""")
    e.indent()
    e.emitln("for lines in fh.readlines():")
    e.indent()
    e.emitln("...")

Given the following file:

myfile.py¶
def some_function():
    # <<? include_file_output "/etc/issue"
    # ?>>

The expanded output would become:

myfile.py¶
def some_function():
    # <<? include_file_output "/etc/issue"
    with open("/etc/issue") as fh:
        for lines in fh.readlines()
            ...
    # ?>>

Note: While the emitter controls the level of indentation, the characters used to indent the line is actually determined by gcgen_indent_by, which can be used to define how indentation should be handled on a file-type basis.

Emitter API¶

class gcgen.api.Emitter(prefix: str, indent_by: str = ' ')

Helper to emit text and control indentation.

Parameters:

  • prefix – base prefix of each line, this will correspond to the level of indentation of the snippet start line or the parent emitter.

  • indent_by – the sequence of characters to use for each level of indentation. E.g. could be 4 spaces for Python files or one tab character for Go files.

add_section() → Emitter

Add section to be filled in later.

Returns a new emitter object, whose contents will form the part of the final output between the prior line of this emitter (if any) and any subsequently added lines.

Note: this function will cause a newline if the current line has any contents already written to it (using emit).

Returns:

A new emitter object whose contents will be injected into the final output at this point

dedent() → None

Dedent lines by one.

Causes all future lines to be indented one level less.

Note: this function will cause a newline if the current line has any contents already written to it (using emit).

emit(*s: str) → None

emit one or more strings to the output.

Emit one or more strings. If multiple strings are provided, they will be concatenated without any space or separation.

Note: in most cases, using emitln with an f-string is a better idea.

Parameters:

*s – one or more strings, which will be concatenated.

emitln(*s: str) → None

Emit a new line to the output.

Writes a full line of output, taking care to properly indent the line as defined by the indent_by attribute and current level of indentation.

Parameters:

*s – one or more strings, each becoming a separate line of output.

emitln_l(*s: str) → None

Emit line, then dedent.

emitln_r(*s: str) → None

Emit line, then indent.

ensure_padding_lines(nlines: int)

Ensure (at least) n empty lines of padding between two sections

NOTE: if the buffer is empty, this call is a NO-OP.

freshline() → None

emit newline iff not currently at the beginning of a new line

indent() → None

Indent subsequent lines.

Causes all future lines to be indented one level more.

Note: this function will cause a newline if the current line has any contents already written to it (using emit).

lines() → Iterator[str]

Return iterator of each element in buffer.

newline() → None

add a newline
